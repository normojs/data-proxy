# 子站功能开发计划

## 背景

子站功能用于在 Data Proxy 主站下创建轻量活动站点/白标入口。子站复用主站账号体系和计费底座，但拥有独立入口、品牌配置、注册登录页、单页用户控制台、专属渠道、额度窗口和日志统计。

V1 目标不是完整多租户 SaaS，而是先交付一个可运营的子站活动控制台：站长可以配置站点、渠道和额度，用户通过子站入口注册/登录、创建 API Key，并在子站控制台查看额度、统计和最近调用。

## 当前状态

- [x] P0 数据模型和权限底座已落地：子站、成员、额度策略、计数器，以及 channel/token/log/billing/capture 的 `subsite_id` 字段。
- [x] P1 子站访问链路已落地：状态解析、公开状态 API、关闭/过期/未开始/不存在响应、关闭页和 `/s/:slug/v1` gate。
- [x] P2-P4 第一个可用闭环已落地：注册登录、成员关系、单页控制台、子站 Key、`/s/:slug/v1`。
- [x] P5-P8 运营闭环已落地：子站渠道、额度结算、管理端、日志审计、request trace 和诊断过滤。
- [x] P9 上线验收已收口：后端测试、前端 smoke、Docker/生产迁移验证和运维文档均有覆盖。

## 范围原则

- [x] 主站账号体系复用，不新增独立用户表。
- [x] 子站访问、API 调用、额度、渠道、日志都必须按 `subsite_id` 隔离。
- [x] 普通子站用户不进入主站 Dashboard，不显示主站菜单。
- [x] 子站站长只能管理自己拥有的子站，主站管理员拥有兜底管理能力。
- [x] 子站渠道密钥加密保存，任何页面不回显完整上游 Key。
- [x] 子站关闭或过期时，网页显示关闭页，API 返回明确错误码。

## 推荐路由

- [x] `/s/:slug`：子站首页/入口，未登录时引导登录或注册。
- [x] `/s/:slug/login`：子站登录页，复用主站登录能力。
- [x] `/s/:slug/register`：子站注册页，按子站注册策略控制。
- [x] `/s/:slug/dashboard`：子站用户单页控制台。
- [x] `/s/:slug/v1`：子站 OpenAI 兼容 API Base URL。
- [x] `/dashboard/subsites`：主站管理员/子站站长的子站管理入口。

## 状态模型

- [x] `enabled`：正常开放。
- [x] `disabled`：手动关闭，网页显示关闭页，API 返回 `subsite_disabled`。
- [x] `draft`：草稿，普通访问不可用，返回 `subsite_draft`。
- [x] `not_started`：不单独落库，由有效期计算得出，网页显示未开始页，API 返回 `subsite_not_started`。
- [x] `expired`：不单独落库，由有效期计算得出，网页显示活动结束页，API 返回 `subsite_expired`。

## 额度模型

子站需要同时支持站点级额度和用户级额度。请求前按最严格规则拦截，请求后按真实消耗结算。

- [x] 子站每日总额度：整个子站每天最多可消耗额度。
- [x] 子站窗口期总额度：例如最近 3 小时最多可消耗额度。
- [x] 用户每日额度：单个用户每天最多可消耗额度。
- [x] 用户窗口期额度：单个用户在当前窗口最多可消耗额度。
- [x] 请求次数限制：可选，支持子站级和用户级窗口限制。
- [x] 窗口期配置：支持固定每日刷新和滚动窗口两类。
- [x] 额度恢复时间：控制台展示下一次恢复/重置时间。

## 开发顺序

### 总体交付顺序

- [x] 1. 先做数据模型和权限底座，保证后续页面、API、账单都有隔离字段。
- [x] 2. 做子站访问状态链路：打开、关闭、草稿、未开始、过期、slug 不存在。
- [x] 3. 做子站注册登录和成员关系，确保子站用户能进入自己的轻量控制台。
- [x] 4. 做子站用户单页控制台，先让用户能看 Base URL、Key、额度和最近请求。
- [x] 5. 做子站 API Key 和 `/s/:slug/v1`，实现真实可调用的子站 API Base URL。
- [x] 6. 做子站渠道和模型开放，站长可以配置自己的上游渠道和模型范围。
- [x] 7. 做额度拦截和结算，保证站点额度、用户额度、窗口期额度都能生效。
- [x] 8. 做子站管理端，把配置、成员、渠道、额度、日志集中到 `/dashboard/subsites`。
- [x] 9. 做日志、审计和诊断，把子站维度接入 usage logs、request trace、诊断包。
- [x] 10. 做上线验收和部署文档，补齐测试、迁移验证、Docker 生产验证。

### P0 数据模型和权限底座

- [x] 新增 `subsites` 表：名称、slug、logo、favicon、title、主题色、公告、状态、关闭原因、有效期、创建者。
- [x] 新增 `subsite_members` 表：`subsite_id`、`user_id`、`role`、`status`、加入时间。
- [x] 新增 `subsite_quota_policies` 表：站点每日、站点窗口、用户每日、用户窗口、请求次数限制。
- [x] 新增 `subsite_quota_counters` 表：站点/用户维度的日窗口和滚动窗口计数。
- [x] `channels` 增加 `subsite_id`，为空表示主站渠道，不为空表示子站专属渠道。
- [x] `tokens` 增加 `subsite_id`，用于绑定子站 Key。
- [x] `logs` 增加 `subsite_id`，用于子站日志查询和统计。
- [x] `billing_events` 增加 `subsite_id`，用于子站账本追踪。
- [x] 增加子站 RBAC helper：主站管理员、子站 owner/admin/member。
- [x] 增加 slug 唯一性校验、保留 slug 校验和软删除策略。

### P1 子站访问链路和关闭页

- [x] 实现子站 resolver：按 `slug` 解析子站，后续支持基础信息缓存和缓存失效。
- [x] 实现子站状态判断：草稿、关闭、过期、未开始、正常。
- [x] 实现公开状态 API：`GET /api/subsites/:slug/public` 返回 logo、title、状态、关闭原因、有效期。
- [x] 子站关闭时拦截 `/s/:slug`、登录、注册和 dashboard 页面。
- [x] 实现关闭页：大图标、子站名称、关闭说明、可选联系方式。
- [x] 子站未开始时显示未开始页，API 返回 `subsite_not_started`。
- [x] 子站过期时网页显示活动结束页，API 返回 `subsite_expired`。
- [x] 子站不存在时显示不存在页面，API 返回 `subsite_not_found`，避免暴露内部错误。
- [x] 子站关闭时 `/s/:slug/v1` 返回 `subsite_disabled`。
- [x] `/s/:slug/v1` 已接入真实 relay，并保留 `subsite_api_not_ready` 作为未实现路径兜底错误码。
- [x] 增加 P1 后端单测：公开状态 API、`/s/:slug/v1` 状态 gate、关闭/过期/不存在错误码。
- [x] 增加 P1 前端 smoke：关闭页、过期页、不存在页移动端和桌面端不重叠。

### P2 子站注册登录和成员关系

- [x] 子站登录页复用主站认证，但登录成功后回到 `/s/:slug/dashboard`。
- [x] 子站注册页复用主站用户创建逻辑，并自动创建 `subsite_members` 记录。
- [x] 支持注册策略：开放注册、邀请码注册、关闭注册。
- [x] 支持邮箱域名白名单。
- [x] 登录后校验用户是否仍是子站成员。
- [x] 被禁用成员访问 dashboard 显示禁用说明。
- [ ] 子站用户退出登录后回到子站入口，而不是主站入口。

### P3 子站用户单页控制台

- [x] 按参考图实现无菜单单页控制台。
- [x] 顶部显示子站 logo、名称、控制台标题和说明。
- [x] API 区显示 Base URL、当前子站 Key、显示/复制/重置按钮。
- [x] 公告区支持站长配置图标、标题、正文和外链。
- [x] 当前套餐/额度区展示每日额度、已用、剩余、下次重置。
- [x] 当前窗口可用量展示站点窗口、用户窗口、请求次数。
- [x] 24 小时统计展示调用次数、Token、费用、最后请求。
- [x] 最近调用记录展示时间、模型、输入、输出、缓存输入、思考 token、总 token、费用、耗时、状态。
- [x] 空状态、加载态、错误态按子站控制台风格实现。
- [x] 移动端控制台保证 API Key、额度卡和日志表可读。

### P4 子站 API Key 和 `/s/:slug/v1`

- [x] 子站控制台支持创建用户自己的子站 Key。
- [x] 子站 Key 绑定 `subsite_id`，不能跨子站使用。
- [x] `/s/:slug/v1` 校验请求 Key 是否属于当前子站。
- [x] Base URL 在控制台显示为当前子站 API 地址。
- [x] 重置 Key 时只影响当前子站 Key。
- [x] 子站 Key 日志写入 `subsite_id`。
- [x] 主站普通 Key 不允许直接访问子站 API，除非管理员显式配置。

### P5 子站渠道和模型开放

- [x] 子站站长可新增、编辑、禁用自己的渠道。
- [x] 子站渠道上游 Key 加密保存，不回显完整 Key。
- [x] 子站渠道只参与本子站 `/s/:slug/v1` 路由。
- [x] 子站站长可选择开放模型列表和展示名称。
- [x] 子站模型列表支持按渠道能力同步。
- [x] 渠道测试和余额检测限定在子站权限内。
- [x] 主站管理员可查看和接管所有子站渠道。

### P6 额度拦截和结算

- [x] 请求前检查子站每日总额度。
- [x] 请求前检查子站窗口期总额度。
- [x] 请求前检查用户每日额度。
- [x] 请求前检查用户窗口期额度。
- [x] 请求前检查请求次数限制。
- [x] 请求完成后按真实消耗写入子站计数器。
- [x] 流式请求失败时按已有账单策略结算或回滚。
- [x] Redis 可用时使用原子计数器，Redis 不可用时降级 DB 事务。
- [x] 额度超限时返回明确错误码：`subsite_quota_exceeded`、`subsite_user_quota_exceeded`、`subsite_rate_limited`。
- [x] 控制台展示预计恢复时间。

- [x] 管理子站渠道和模型开放。
- [x] 查看子站日志、用量统计和错误请求。

### P8 日志、审计和排障

## V1 不做

- [ ] 子站自定义域名。
- [ ] 子站独立支付/分佣。
- [ ] 子站复杂多菜单后台。
- [ ] 子站独立账号体系。
- [ ] 子站跨服务器多节点配额协调。
- [ ] 子站独立主题模板市场。

## 风险点

- [x] 额度计数必须和真实扣费保持一致，避免窗口额度显示与实际账单不一致。
- [x] 子站渠道 Key 必须防止被站长以外的人读取。
- [x] `/s/:slug/v1` 和 `/v1` 的路由上下文必须隔离，避免主站 Key 混用子站额度。
- [x] 日志和账单必须带 `subsite_id`，否则后续统计和追责会困难。
- [x] 子站关闭时要同时拦截页面和 API，不能只隐藏前端入口。
